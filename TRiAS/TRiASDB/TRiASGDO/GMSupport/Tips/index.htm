Koordinatensystem für eine Arc-Info Datenquelle festlegen

Der Arc-Info Datenserver für GDO benötigt Informationen über das Koordinatensystem in dem die Arc-Info Daten abgelegt. Diese Informationen sind nicht implizit in den Daten abgelegt sondern müssen dem Datenserver explizit mitgeteilt werden. Hierbei gilt es zwei Fälle zu unterscheiden:

Fall 1 - Die Koordinaten sämtlicher Layer / Objektklassen liegen in einem Koordinatensystem vor

In diesem Fall ist es ausreichend im Arc-Info Datenverzeichnis eine .ini - Datei mit folgendem Inhalt zu erstellen:

Das gezeigte Beispiel bezieht sich auf die oben angegebene Verzeichnisstruktur. In dieser .ini Datei wird dem Arc-Info Datenserver mitgeteilt wo er die für das Projekt zuständige Koordinatensystemdefinition finden kann. Diese .csf Datei kann mit Hilfe des bei Geomedia mitgelieferten Tools erstellt werden. Nachdem Sie die .ini Datei und die Koordinatensystemdefinitionsdatei erstellt haben, sollte Ihr Arc-Info Datenverzeichnis also wie folgt aussehen:

Hinweis: Zum Definieren der .ini- und der Koordinatensystmdefinitionsdatei sollten sie das Programm GeoINIwz.exe von Intergraph benutzen.

Fall 2 - Die Koordinaten der Layer / Objektklassen liegen in unterschiedlichen Koordinatensystemen vor

In diesem Fall ist für jeden Layer eine Koordinatensystmdefinitionsdatei zu erstellen. Diese Datei muß im Verzeichnis des Layers liegen, den Namen des Verzeichnisses haben. Eine .ini Datei ist in diesem Fall nicht notwendig, weder im Arc-Inf Datenverzeichnis noch in den einzelnen Layerverzeichnissen. Das Tool zum definieren einer Geomedia-Koordinatensystemdatei finden sie unter Startmenü->Programme->Koordinatensystem definieren.
